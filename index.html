Programming languages

  • Ones and Zeros

    Instructions written in a formal artifical language to manipulate the ones and zeros which are the language of the computer hardware.

    As scripts or text files that the computer understands.

  • Artificial Language

    Albeit artificial they are constructed with our own 'natural' language in mind.

  • Speaking your language

Interrupt

  • Traditional computing: Start to Finish

    Pre 1950s, programming computers consisted of writing instructions (scripts) and feeding them to the computer (executing) to run from start to finish. No intervention during this time.

    No intervention until the script has finished.

  • New paradigm: Interactive Computing

    In the 1950s a new paradigm emerged that allowed for scripts to be interrupted and for interaction to occur between the computer and external world.

  • Peripheral (External) Devices

    Keyboards, mice, network cards, microphones, sounds sensors, light sensors, switches, etc

Scripts (Sketches) and Execution

  • Scripts (Sketches)

    A text file containing the code, which are the instructions to be followed during execution.

  • Execution / Running

    Alone the text file is redundant. We execute or run these instructions.

  • Arduino

    The Ardunio microcontroller is where the execution occurs. It contains one script (Sketch) and continually executes this for us.

Comments

  • Note to self and others

    A programmers best friend. Written within a script but completely ignored during execution.

    A footnote. Showing your working. A reminder. A message to others who might encounter your code.

Variables

A way to manage the data and values within our scripts and applications.

  • Container for a value

    A variable contains a chunk of data of specific type.

  • Meaningful name

    It should be given a unique and meaningful name to help you identify the data or value stored within. Usually named as a noun.

  • Varying variables

    They can and usually do contain changing (varying) data.

Variables: Arrays

  • Collection of values

    A variable that can hold numerous values that are set and accessed by their position in the array.

  • Square brackets

    Identified as an array by the square brackets after name.

  • Starts from 0

    Values in an array are accessed using their position, which starts from zero.

Square brackets to identify variable as an array. The type of values to be held in array. Array values container in curly brackets and each value defined by a comma. 0 index.

Functions / Subroutines

  • A collection of code

    A collection (or block) of code that performs a specific task.

    A way to organise your code. A subprogram or program within a program.

  • Reusable

    Code is usually put into a function so that it can be started (called) several times.

  • Meaningful name

    Like a variable, should be given a unique and meaningful name to help you identify the purpose of the code within. Usually named as a verb.

It is defined in one place and then called in another. The code is contained within curly braces/brackets.

Conditionals: if-then-else

  • Controls the flow and ouput of script

    Used to control the flow and output of a running script based around the values stored in variables.

  • Comparing Variables: if() statement

    Using the if() statement you compare the values within variables using equal to(==), not equal to(!=), greater than(>) and lesser than(<).

  • The expressions within the brackets will be either true or false.

  • If the expression is true then the code after the if() will run. If the expression is false the code after the else will run.
